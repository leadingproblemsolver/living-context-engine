# Validation Readiness Report

## Technically ready when

- `npm run ship` passes;
- auth-free migration is applied;
- function is deployed with JWT verification disabled and allowed origins set;
- private-browser smoke test completes the full path;
- one run is consumed exactly once per submitted checkpoint;
- refresh restores same-browser history;
- direct table writes fail.

## Human-side ready when

- outreach targets have a live or recent consequential incident decision;
- the invitation asks for one real checkpoint, not generic feedback;
- the user understands redaction and responder-authority boundaries;
- the founder records whether the checkpoint changed the decision, test, timing, or participants;
- follow-up requests the outcome and a second run only after first value.

## Validation gates

- 3 non-demo submissions within 24 hours;
- 2 completed explicit next moves;
- 1 decision/test/timing/participant change or materially strengthened decision;
- 1 executable falsification check actually run;
- 1 export/share or second run;
- after 5 real checks, at least 50% decision-impact rate before expanding scope.
